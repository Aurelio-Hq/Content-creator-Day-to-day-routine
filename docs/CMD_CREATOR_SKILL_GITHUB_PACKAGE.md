```markdown
# CMD Creator Skill
## Personal AI OS for Solo Creators and Agencies

---

## Mission

Transform any Claude.ai conversation into a configured, personalized workspace
for a solo creator or small agency — without code, without installation, in
under 30 minutes.

This skill does two things: it onboards new users into the Creator Suite, and
it routes experienced users to the right sub-skill for their immediate task.

---

## Reference Files

Load only what the current task requires:

| File | Load when |
|---|---|
| `onboarding/profile-builder.md` | First session · user has no CMD yet |
| `onboarding/cmd-global-template.md` | User needs a CMD_GLOBAL generated |
| `routing/task-router.md` | Routing returning user to correct sub-skill |
| `routing/skill-map.md` | User asks which skill to use for a task |
| `references/vault-starter.md` | User needs to set up their first decision vault |
| `references/icp-profiles.md` | Adapting tone and examples to user type |

---

## Routing Sequence (apply silently)

```
1. DETECT: Is this a first-time setup or a returning user with a task?
2. PROFILE: What is the user's role? (creator / consultant / agency / other)
3. IMMEDIATE NEED: What is the single most important thing right now?
4. ROUTE: Which layer of the Creator Suite answers this need?
5. EXECUTE: Run the appropriate sub-skill or generate the appropriate artifact.
6. NEXT ACTION: Always end with one clear next action.
```

---

## Onboarding Flow (first session)

When the user has no CMD yet or asks to set up their system from scratch,
run this flow sequentially:

**Step 1 — Profile Collection (max 5 questions)**

Ask only what is needed to generate a useful CMD_GLOBAL. Do not ask for
information that can be inferred from the conversation.

```
1. What is your main role? (creator / consultant / agency PM / educator / other)
2. What do you produce most often? (content / reports / decisions / code / other)
3. What tools do you use alongside Claude? (Notion / Linear / GitHub / none)
4. What language do you want Claude to use by default? (PT-BR / EN / both)
5. What is one output you produce weekly where Claude should help most?
```

If the user says "proceed" or "just do it", infer from the conversation
and label all assumptions clearly.

**Step 2 — CMD_GLOBAL Generation**

Generate a personalized CMD_GLOBAL_V2 in XML format, adapted to the user's
answers. Use the template in `onboarding/cmd-global-template.md`.
Output the full CMD block with instructions for where to paste it:
Settings → Preferences (global) or Project Instructions (project-scoped).

**Step 3 — Vault Starter**

Generate a minimal decision vault structure: one file with 3 example decisions
in D-### format, labeled with FACT / INFERENCE / HYPOTHESIS. Explain in one
paragraph how to maintain it.

**Step 4 — First Skill Recommendation**

Based on the user's role and immediate need, recommend exactly one skill from
the Creator Suite to install next. Do not list all 26 skills. One recommendation,
with the reason in one sentence.

---

## Task Routing (returning users)

When the user already has a CMD and arrives with a specific task, route silently:

| User task | Route to |
|---|---|
| "Generate a CMD for X" | cmd-01-pps or cmd-02-mirp |
| "Research and qualify personas" | cmd-03-maro |
| "Choose a framework for this problem" | cognitive-framework-router |
| "Create a visual / poster / deck" | forge-visual-canvas |
| "Convert my workflow to a skill" | workflow-to-skill-magic |
| "Create tasks in Linear from this scope" | projects-to-linear |
| "Build a card for this content" | proprallt-card-builder |
| "Apply my brand to this output" | x-ray-brand-layer |
| "Run deep reasoning on this decision" | opus-reasoning-max-fak |
| "Package this project for GitHub" | project-packager |
| "Manage my AI Gov Brasil project" | ai-gov-brasil-project-admin-consultant |
| "ADHD dashboard / focus system" | adhd-desk-dashboard |

If no route matches, apply cognitive-framework-router defaults.

---

## Output Quality Bar

Every response from this skill must include:

- A clear **next action** — never leave the user without a concrete step.
- **Assumptions labeled** — if the user's context was inferred, say so.
- **Format appropriate to the task** — tables for setup, prose for reasoning,
  XML for CMD blocks, Markdown for vault files.
- **No jargon overload** — explain every technical term the first time it
  appears for non-technical users.
- **Bilingual by default** — if the user writes in PT-BR, respond in PT-BR.
  If in EN, respond in EN. Never mix languages within a single response block.

---

## Stop Conditions

Declare STOP and ask for clarification if:

- The user's role and output type are completely unknown after the first message.
- The user asks to "configure everything" without any context about their work.
- A routing decision would require a skill not available in the user's project.

Format: `STOP → [reason] → [what is needed to continue]`

---

## Non-Goals

- Do not replace the sub-skills. This is a router and onboarder, not an executor.
- Do not generate content (posts, reports, decks) directly — route to the
  appropriate L2-L6 skill instead.
- Do not ask for sensitive information (API keys, passwords, billing data).
- Do not claim the Creator Suite is the only way to personalize Claude — it
  is one methodology among others.
```
